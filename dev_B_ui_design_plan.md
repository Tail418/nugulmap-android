# 개발자 B: UI/UX 디자인 계획 (수정안)

이 문서는 `ui_idea` 프로토타입 분석 결과를 반영하여 기존 UI/UX 디자인 계획을 수정한 최종안입니다.

## 1. 🎯 핵심 원칙 (Core Principles)

-   **명확성과 집중 (Clarity & Focus):** 지도 중심의 앱이므로, 불필요한 UI 요소를 최소화하고 지도와 핵심 정보(흡연구역)에 사용자의 시선이 집중되도록 설계합니다.
-   **직관적 상호작용 (Intuitive Interaction):** 사용자가 지도 탐색, 정보 확인, 구역 제보 등 핵심 기능을 별도의 학습 없이 쉽고 빠르게 사용할 수 있도록 직관적인 UI를 제공합니다.
-   **일관성 (Consistency):** 앱의 모든 화면에서 일관된 색상, 타이포그래피, 컴포넌트 스타일을 유지하여 통일감 있고 안정적인 사용자 경험을 제공합니다.

## 2. 🎨 색상 및 타이포그래피 (Color & Typography)

-   **색상 (Color):**
    -   **모노크롬 테마:** `ui_idea/frontend/app/globals.css`를 기반으로, **흰색(60%), 회색(30%), 검은색(10%)**을 중심으로 한 미니멀한 모노크롬 색상 팔레트를 사용합니다.
    -   **다크 모드 미지원:** 초기 버전에서는 일관되고 단순한 사용자 경험을 위해 라이트 모드만 지원합니다.
    -   **소셜 로그인 버튼:** 카카오(노란색), 네이버(녹색) 등 각 브랜드의 고유 색상을 사용하여 사용자가 쉽게 인지할 수 있도록 합니다.

-   **타이포그래피 (Typography):**
    -   **기본 폰트:** 가독성이 뛰어난 `Kakao` 웹 폰트를 앱 전체의 기본 글꼴로 사용합니다.
    -   **로고 폰트:** `LoginPage`에서 확인된 바와 같이, 앱의 영문 로고(`NeogulMap`)에는 `Righteous` 폰트를 사용하여 디자인적 포인트를 줍니다.
    -   **위계질서:** `MaterialTheme.typography` 스펙을 참고하여 제목(`headlineSmall`), 본문(`bodyLarge`) 등 텍스트 스타일을 명확히 구분하여 사용합니다.

## 3. 📱 화면별 레이아웃 및 컴포넌트 (Layout & Components)

#### `LoginPage` (로그인 화면)

-   중앙에 배치된 `Card` 컴포넌트 안에 로고 이미지(`pin.png`), `Righteous` 폰트를 사용한 앱 이름, 그리고 카카오/네이버/구글 소셜 로그인 버튼을 명확하게 배치합니다.

#### `TermsPage` (약관 동의 화면)

-   로그인 후, 프로필 설정 전 단계로 '전체 동의' 및 서비스 이용약관, 개인정보 처리방침(필수), 마케팅 정보 수신(선택) 항목을 포함하는 약관 동의 화면을 제공합니다. 각 약관 내용은 `ScrollArea` 컴포넌트 내에 표시합니다.

#### `SignupPage` (프로필 설정 화면)

-   약관 동의 후, 닉네임과 프로필 이미지를 설정하는 화면입니다. `Avatar` 컴포넌트를 통해 이미지 업로드 및 미리보기를 지원하고, 기본 이미지로 대체할 수 있는 옵션을 제공합니다.

#### `HomeScreen` (홈 - 지도 화면)

-   **레이아웃:** 화면 전체를 지도로 채우고, 주요 기능들은 지도 위에 플로팅(floating) 형태로 배치합니다.
-   **주요 컴포넌트:**
    -   **`SearchBar` (검색 바):** 상단 중앙에 위치.
    -   **`FloatingUserProfile` (프로필 버튼):** 상단 우측에 위치하며, 클릭 시 드롭다운 메뉴(내 정보, 설정, 로그아웃)를 표시합니다.
    -   **`CurrentLocationButton` (현재 위치 버튼):** 하단 좌측에 위치.
    -   **`FloatingActionButton` (제보 버튼):** 하단 우측에 위치하며, '+' 아이콘을 통해 신규 흡연구역 등록 모달(`AddLocationModal`)을 엽니다.
    -   **흡연구역 상세 정보:** 마커를 클릭하면 화면 하단에 `Card` 형태의 정보창이 나타나며, 선택된 구역의 이미지와 주소, 타입 등의 요약 정보를 보여줍니다. (기존 `BottomSheet` 안에서 `AlertDialog` 대신 `Card` UI를 채택)

#### `AddLocationModal` (신규 구역 등록 모달)

-   `FloatingActionButton` 클릭 시 나타나는 `Dialog` (모달) 형태입니다.
-   주소 검색, 지도에서 위치 확인, 구역 타입/크기 선택(`Select`), 상세 설명(`Textarea`), 이미지 업로드 기능을 포함한 폼으로 구성됩니다.
-   모달이 열릴 때 사용자의 현재 위치를 가져와 주소 필드를 자동으로 채워주는 UX를 제공합니다.

#### `ProfilePage` (프로필 화면)

-   **탭 기반 레이아웃:** `Tabs` 컴포넌트를 사용하여 '프로필 정보'와 '내 등록 장소'를 분리합니다.
-   **프로필 정보 탭:** 닉네임 편집, 프로필 이미지 변경 기능을 제공합니다.
-   **내 등록 장소 탭:** 사용자가 등록한 장소 목록을 `Card` 리스트 형태로 보여줍니다.

## 4. ✨ 애니메이션 및 마이크로 인터랙션 (Animation & Interactions)

-   **`tailwindcss-animate` 활용:** `slide-in-from-bottom`, `zoom-in` 등 `v0.app`에서 사용된 애니메이션을 적극적으로 활용하여 UI에 동적인 느낌을 더합니다.
-   **로딩 상태:** 데이터 로딩 시에는 `Loader2` 아이콘과 같은 스피너를 표시하여 사용자에게 시스템이 작동 중임을 명확히 알립니다.
-   **피드백:** 버튼 클릭 및 폼 제출 성공 시 `Check` 아이콘을 표시하는 등, 사용자 행동에 대한 시각적인 피드백을 제공합니다.
